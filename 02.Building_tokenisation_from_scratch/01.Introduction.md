# Tokenisation: A Detailed Overview

## What is Tokenisation?

Tokenisation is the process of breaking down text into smaller units called **tokens**. These tokens can be words, subwords, or characters, depending on the chosen tokenisation strategy. It's the **first and most fundamental step** in preparing text data for natural language processing (NLP) models.

## Why is Tokenisation Useful?

### 1. **Bridge Between Raw Text and Numerical Data**
   - NLP models (neural networks, etc.) understand numbers, not text
   - Tokenisation converts human-readable text into machine-understandable numerical representations

### 2. **Handles Vocabulary Limitations**
   - Models have fixed vocabulary sizes
   - Tokenisation manages out-of-vocabulary (OOV) words by breaking them into known sub-tokens

### 3. **Standardises Input**
   - Handles variations in spacing, punctuation, and formatting
   - Normalises text for consistent processing

### 4. **Enables Efficient Processing**
   - Reduces computational complexity by working with token IDs instead of raw strings
   - Facilitates batch processing with uniform input sizes

### 5. **Preserves Linguistic Information**
   - Different tokenisation strategies can preserve different levels of linguistic structure

## The Complete Tokenisation Pipeline

### **Step 1: Text Normalization**
Before splitting, we clean and standardize the text:
```python
# Example normalization steps
text = "Hello, World! Let's tokenize this text."
# Lowercasing
text = text.lower()  # "hello, world! let's tokenize this text."
# Remove excessive whitespace
text = ' '.join(text.split())  # Standardize spacing
```

### **Step 2: Splitting into Tokens**
The actual splitting process depends on the tokenisation strategy:

#### **Word-Level Tokenisation**
```python
# Simple whitespace splitting
text = "I love NLP"
tokens = text.split()  # ["I", "love", "NLP"]

# With punctuation handling
import re
tokens = re.findall(r"\b\w+\b", text)  # ["I", "love", "NLP"]
```

#### **Subword Tokenisation** (Most Common in Modern NLP)
- Splits rare/unknown words into known subwords
- Popular algorithms: BPE, WordPiece, Unigram, SentencePiece

**Byte Pair Encoding (BPE) Example:**
```
Vocabulary: ["low", "lower", "newest", "widest"]
Training corpus: "low lower newest widest"

Step 1: Start with character vocabulary: {l, o, w, e, r, n, s, t, i, d}
Step 2: Count character pairs: "lo": 2, "ow": 2, ...
Step 3: Merge most frequent pair: "lo" → "lo"
Step 4: Continue merging: "low" appears frequently → add to vocabulary
```

### **Step 3: Assigning Token IDs**
Each token gets a unique numerical ID:

```python
# Vocabulary mapping
vocab = {
    "[PAD]": 0,    # Padding token
    "[UNK]": 1,    # Unknown token
    "[CLS]": 2,    # Classification token
    "[SEP]": 3,    # Separator token
    "hello": 4,
    "world": 5,
    "let": 6,
    "'s": 7,
    "token": 8,
    "##ize": 9,    # Subword prefix (in WordPiece/BERT)
    "this": 10,
    "text": 11,
    ".": 12
}

# Convert tokens to IDs
tokens = ["hello", "world", "!", "let", "'s", "token", "##ize", "this", "text", "."]
token_ids = [vocab.get(token, vocab["[UNK]"]) for token in tokens]
# Result: [4, 5, 1, 6, 7, 8, 9, 10, 11, 12]
```

### **Step 4: Creating High-Dimensional Embedding Vectors**

#### **What are Embeddings?**
- Dense vector representations of tokens in a continuous vector space
- Typically 128-1024 dimensions
- Capture semantic relationships (similar words have similar vectors)

#### **Embedding Process:**
```python
import torch
import torch.nn as nn

# Parameters
vocab_size = 10000  # Size of vocabulary
embedding_dim = 512  # Dimension of embedding vectors
max_seq_length = 128  # Maximum sequence length

# Create embedding layer
embedding_layer = nn.Embedding(
    num_embeddings=vocab_size,
    embedding_dim=embedding_dim,
    padding_idx=0  # Index for padding token
)

# Convert token IDs to embeddings
token_ids = torch.tensor([4, 5, 1, 6, 7, 8, 9, 10, 11, 12])
embeddings = embedding_layer(token_ids)

# embeddings shape: [10, 512]
# Each token is now represented as a 512-dimensional vector
```

#### **How Embeddings Capture Meaning:**
```
Word: "king"
Embedding: [0.2, -0.1, 0.5, 0.3, -0.4, ...]  # 512 values

Word: "queen"
Embedding: [0.19, -0.08, 0.51, 0.28, -0.38, ...]  # Similar vector

Word: "apple"
Embedding: [-0.3, 0.6, 0.1, -0.5, 0.2, ...]  # Different vector
```

## Practical Example: Full Tokenisation Pipeline

```python
from transformers import AutoTokenizer
import torch

# 1. Load a pre-trained tokeniser
tokenizer = AutoTokenizer.from_pretrained("bert-base-uncased")

# 2. Input text
text = "Hello, World! Let's tokenize this text."

# 3. Tokenisation (Steps 1-3 combined)
encoding = tokenizer(
    text,
    truncation=True,
    padding="max_length",
    max_length=128,
    return_tensors="pt"
)

# What happens inside:
# - Normalization: lowercasing, punctuation handling
# - Tokenisation: WordPiece algorithm
# - Special tokens: [CLS] at start, [SEP] at end
# - Token IDs: Convert tokens to numbers

print("Token IDs:", encoding["input_ids"])
# Example: [101, 7592, 1010, 2088, 999, 102, ...]

print("Tokens:", tokenizer.convert_ids_to_tokens(encoding["input_ids"][0]))
# Example: ['[CLS]', 'hello', ',', 'world', '!', '[SEP]', ...]

# 4. Embedding (Step 4)
# In practice, the model handles this automatically
model_input = {
    "input_ids": encoding["input_ids"],
    "attention_mask": encoding["attention_mask"]
}
```

## Advanced Tokenisation Techniques

### **1. Byte-Level BPE** (Used in GPT models)
- Treats text as sequence of bytes
- Handles any Unicode character
- Vocabulary size typically 50,256

### **2. SentencePiece**
- Language-agnostic tokenisation
- Treats text as raw Unicode
- Doesn't require pre-tokenisation

### **3. WordPiece** (Used in BERT)
- Similar to BPE but uses likelihood, not frequency
- Merges based on maximizing language model likelihood

## Common Challenges and Solutions

### **Challenge 1: Out-of-Vocabulary Words**
**Solution:** Subword tokenisation
```
"unhappiness" → ["un", "##happi", "##ness"]
```

### **Challenge 2: Multiple Languages**
**Solution:** Multilingual tokenisers or language-specific tokenisation

### **Challenge 3: Domain-Specific Terms**
**Solution:** 
- Domain-adaptive tokenisation
- Extend vocabulary with domain terms

### **Challenge 4: Sequence Length Limitations**
**Solution:** 
- Truncation strategies
- Sliding window approaches
- Hierarchical models

## Best Practices

1. **Choose the Right Tokeniser for Your Model**
   - Use the same tokeniser the model was trained with
   - Ensure consistent pre-processing

2. **Handle Special Tokens Properly**
   - Include [CLS], [SEP], [PAD], [UNK] as needed
   - Positional embeddings for transformer models

3. **Consider Sequence Length**
   - Balance between information retention and computational efficiency
   - Typical lengths: 128-512 tokens

4. **Monitor Vocabulary Coverage**
   - Check percentage of OOV tokens
   - Consider vocabulary extension for domain-specific tasks

## Conclusion

Tokenisation transforms raw text into a structured numerical format that machine learning models can process. The journey from text to embeddings involves:

1. **Normalization** → Cleaning and standardizing text
2. **Splitting** → Breaking text into meaningful units
3. **Token ID Assignment** → Creating numerical representations
4. **Embedding** → Converting to high-dimensional vectors that capture semantic meaning

Modern NLP systems use sophisticated subword tokenisation algorithms (BPE, WordPiece, SentencePiece) that balance vocabulary efficiency with the ability to handle novel words. This fundamental process enables everything from sentiment analysis to machine translation, making it the cornerstone of modern natural language processing.
